# Composition Patterns & Metaphor Rules

## Core Structural Patterns

Select only one primary structural pattern per illustration. Do not overcomplicate by mixing too many.

### Workflow Process

- **Best for:** Input $\rightarrow$ Processing $\rightarrow$ Output, content creation, AI pipelines, automation steps.
- **Visual Layout:** Left side displays inputs, middle shows Desi or an absurd machine processing them, right side shows outputs. Use orange arrows to denote the main direction of the flow.

### System Component

- **Best for:** Information sources, filters, databases, renderers, or specific parts of an agentic system.
- **Visual Layout:** Draw only 3–5 core components or modules, placing Desi at the center of one critical action/interaction.

### Before/After Comparison

- **Best for:** Chaos vs. Order, Manual vs. Automated, Scattered vs. Consolidated, Stressed vs. Relaxed.
- **Visual Layout:** Left side depicts the chaotic/manual state, right side shows the organized/stable state, connected by a prominent middle orange arrow. Characters can be drawn in more exaggerated poses here.

### Character State

- **Best for:** User friction points, creator blocks, tool fatigue, information overload, or moving from stuck to running.
- **Visual Layout:** 2–4 small snapshots of Desi in various states, each accompanied by a short annotation label.

### Concept Metaphor

- **Best for:** Content factories, information warehouses, black-box processing, automation pipelines, automated report generators.
- **Visual Layout:** A large, bizarre physical object or machine with a few inputs and a single output. Needs to be highly memorable.

### Method Layering (Framework Stack)

- **Best for:** Methodological frameworks, system hierarchies, capabilities/tech stacks, layered content systems.
- **Visual Layout:** A stack of wobbly, hand-drawn layers or boxes (avoid formal grid-like pyramid diagrams). Show Desi building, climbing, or bricklaying at the side.

### Map & Path

- **Best for:** Idea-to-launch journey, user journey, content distribution paths, learning roadmaps.
- **Visual Layout:** A wobbly, winding path with 3–5 key milestones or nodes, showing Desi guiding a rope or walking along it.

### Strip Comic Panels

- **Best for:** Failure-to-success steps, user stories, satirical comparisons, before-and-after transitions.
- **Visual Layout:** 2–4 clean, small panels, each showing a single key action.

## Technical System Metaphors

When illustrating technical architectures or pipelines (like RAG systems, vector search, database replication, API endpoints, or model training), do not draw a single generic box or machine. Instead, break the architecture down into its actual sequential stages and map each step to a wobbly, low-tech physical counterpart:

### Technical-to-Physical Mapping Table:

| Technical Concept | Wobbly Physical Metaphor | Example Action |
| :--- | :--- | :--- |
| **Ingestion / Data Loading** | Funnel, wobbly chute, sorting tray, mailbox slot, or hopper container. | Desi pouring papers/files into a wobbly funnel. |
| **Chunking / Splitting** | Bread slicer, guillotine paper cutter, scissors, or cutting board. | Desi slicing long document rolls into short strips. |
| **Embedding / Vectorization** | Ink stamp machine, scanner lens, magnifying glass, or a labeling gun. | Desi stamping geometric shapes onto papers. |
| **Vector DB / Vector Storage** | Multi-drawer filing cabinet, catalog box, shelving unit with labeled grid codes. | Desi sliding shape-labeled files into a wobbly drawer cabinet. |
| **Retrieval / Vector Search** | Magnifying glass search, fishing rod, manual crane/pulley, or card index scan. | Desi using a fishing hook to catch a specific shape block matching the query. |
| **Generation / LLM Output** | Wobbly typewriter, manual printing press, or mechanical mixer drum. | Desi turning a hand crank to output final wobbly folder. |
| **Cache / Memory Store** | Clothesline, temporary desk tray, or a small bucket hanging on the side. | Desi hanging frequently used pages on a clothesline for quick access. |

### Layout Rule:
Connect these steps sequentially from left to right using wobbly orange flow arrows, showing a complete end-to-end pipeline (e.g., Ingestion -> Chunking -> Embedding -> Storage -> Retrieval -> Generation) so the drawing is rich in subject-matter details and explains the exact technical flow.

## Designing Original Metaphors

Always invent a fresh physical metaphor based on the article's text; do not copy previous example layouts.

### The Three-Step Metaphor Process

1. **Convert abstract concepts into physical actions:** getting stuck, leaking, getting heavier, sorting, fermenting, opening doors, folding, unpacking, loops/backflows.
2. **Convert the system structure into a low-tech physical object:** broken machines, cardboard boxes, drawers, water pipes, mailboxes, weird dial panels, weighing scales, water wells, ladders, manual pulleys, weird desks.
3. **Assign Desi a critical action:** Instead of standing in a corner, Desi should be stuck inside the gears, pulling the wrong wires, guarding the gates, carrying blocks, repairing leaks, weighing concepts, holding a ladder, or stamping inputs.

### Physical Object Library (Examples)

- Cardboard boxes, drawers, vintage gears, funnels, balance scales, mailboxes, gates, wells, ladders, plumbing pipes, yarn balls, valves, rotary dials, black boxes, punch-card machines, rolling pin pressers, clotheslines, strange workshop desks.
- Use only 1 or 2 objects in a single image; keep it clean.

### Desi Action Library (Examples)

- Pulling, carrying, stuffing, scooping, pressing, weighing, sewing, cutting, tightening, guarding, pushing, catching, dismantling, stamping, recycling.
- Every action must directly serve the core meaning of the drawing.

## Anti-Plagiarism & Replication Rules

Do not automatically open or replicate the exact compositions in `assets/examples/`. Those files serve only as visual calibrations for lines, whitespace, colors, and Desi's personality.

Unless the user explicitly requests to "modify/replicate this specific layout", do not copy these exact historical layouts:
- Conveyor belt with two distinct breakpoints.
- Desi inside a content machine pulling a judgment lever.
- Desi acting as a funnel to sort traffic/trust/conversion.
- Desi slicing up a fish representing "content assets".
- Desi holding the guide rope of a winding path.
- Desi pulling three ropes representing multiple info inputs.
- Three Desis holding a megaphone, building a bridge, and opening a door.
- Desi stamping words from an "expression toolkit".
- Desi holding warning signs along a path with holes.

Even if you encounter similar themes, invent a new metaphor. For instance, instead of drawing a path for a "journey," draw Desi connecting a wire to a lightbulb; instead of a fish for "multi-purpose content," draw Desi pressing a single lump of clay into different molds.
